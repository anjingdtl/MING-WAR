# 万历：山河崩塌 — UI/UX 优化方案 v1.0

> **对标：Victoria 3** — 在复杂度与可读性之间找到属于明代的平衡
>
> 本文档聚焦**玩家视角的交互体验**与**视觉语言**,不改动 `core/` 下的数值与模拟逻辑。
> 配合 `docs/strategy-layer-optimization-plan.md` 一起读,形成"内功+外形"双轨。

---

## 〇、写在前面：向 V3 学什么

P 社官方公布的 V3 UI 设计哲学(来自 Dev Diary #29、#30、#159)有三个核心支柱,值得我们**直接借骨架、明代化血肉**:

| V3 支柱 | V3 的做法 | 我们怎么本土化 |
|---|---|---|
| **Prestigious 华丽** | 受维多利亚时代上层阶级启发,优雅精致 | 受明代士大夫审美启发 — 官造器物、雕版印刷、文人书房 |
| **Vintage & Idyllic 复古田园** | 浪漫主义 Art Nouveau、植物曲线 | 借鉴**明代卷草纹、如意云头、雕花窗格** — 不走满清繁复、不走宋代极简,取晚明书房雅趣 |
| **Detailed yet Approachable 详细而平易** | 装饰聚焦在**框架与边框**,内容区干净 | 标题、面板边角做精,内部数据区保持高对比与可扫读 |

> V3 的教训也照搬:**"信息密度极高但玩家仍能上手"**靠的不是堆砌,而是**组织 + 分类 + 预测**。

## 一、现状诊断（与 V3 的差距地图）

### 1.1 已经在做的（不要重做）
- ✅ 米黄/古铜色主调、纸面背景纹理（已有"古卷"氛围）
- ✅ 6 个地图图层切换、缩放/拖拽、`FocusTooltip` 派系反应预测
- ✅ 关键人物肖像 sprite（4 类:皇帝/重臣/将军/可汗）
- ✅ 事件插画（3 类:朝堂/边疆/饥荒）按事件分类切换
- ✅ 朝堂派系的颜色编码（支持度>60 绿、40-60 黄、<40 红）
- ✅ 决策控件有清晰的"player-only / 非玩家势力"区分

### 1.2 核心痛点（按优先级）

| # | 痛点 | 严重度 | V3 是怎么解决的 |
|---|---|---|---|
| 1 | **没有 Lens 系统** —— 6 个图层是平铺的,玩家要自己切 | 🔴 高 | V3 的 **5 个 Lens**(经济/政治/军事/人口/外交)是核心范式,每个 Lens 自带地图色板+面板 Tab+数据排序 |
| 2 | **危机信息裸奔** —— `粮尽/破产/叛乱` 全部要等月报才知道,没有任何**预警** | 🔴 高 | V3 地图上色块透明度直接反映控制度,TopBar 的"标准"和"合法度"有独立进度条+趋势箭头 |
| 3 | **Tooltip 仅有一处** —— `FocusTooltip` 写得很漂亮但其他数字全是裸值 | 🔴 高 | V3 几乎所有数字、图标、地名都可 hover 出 tooltip,统一外观 |
| 4 | **Lucide 线条图标滥用** —— 地图上的麦穗、刀剑、税、地形…全是千篇一律的现代线条 | 🟠 中 | V3 按"详细插画/实体物/简化色块"分三类,不同语境用不同精度 |
| 5 | **面板层级混乱** —— `RegionPanel.tsx` 写好了但 App 没用,所有区域详情都塞在 `map-command-panel` 里 | 🟠 中 | V3 用左侧侧滑面板(Side Panel)+ Tabs,所有详情都走同一通道 |
| 6 | **没有 Tutorial/新手引导** | 🟠 中 | V3 Dev Diary #30 明确写:"报纸+蓝图+老旧纸张"风的信息图 |
| 7 | **叙事性缺失** —— `LogPanel` 是流水账,没有"时代感" | 🟡 低 | V3 的 Journal 系统(JOURNAL ENTRIES)是核心叙事通道,玩家主动打开读 |
| 8 | **没有键盘快捷键** | 🟡 低 | V3 大量快捷键:Space 推进、Lens 数字键 1-5、Esc 关闭等 |
| 9 | **没有动效/过渡** —— 数字突变、面板进出都"啪"地一下 | 🟡 低 | V3 数值滚动、面板滑入/淡出、地图色块渐变 |
| 10 | **没有外交/战争详情界面** | 🟡 低 | V3 的 Diplomatic Play 流程、战争进展条是核心交互 |

### 1.3 数据维度 vs. 表现力差距

游戏实体已经够丰富(15+ 字段的 Faction、11 字段的 Region、6 字段的 War、5 字段的 Clique),但**只有约 40% 的字段被 UI 表达出来**。这一份 spec 至少要把覆盖率提到 **75%**。

## 二、设计愿景

> **目标一句话**:让玩家在 5 分钟内就能"一眼看出哪里出事了",在 1 小时内能"像读《万历邸报》一样爱上这个时代"。

三条原则:
1. **信息密度向上、可读性不降** — 不靠加色块,靠**结构化分组**
2. **预测优于结果** — 让玩家在**点击前**就能看到后果(扩展已有的 `FocusTooltip` 思路)
3. **时代感一致** — 所有视觉元素都从明代实物/审美取材,杜绝"游戏 UI 通用模板"

## 三、视觉系统重构

### 3.1 配色：明代官造色 + 朝代语义

| 角色 | 色名 | HEX | 用途 |
|---|---|---|---|
| 主色-朱 | **帝王朱** | `#B23A2C` | 关键行动按钮、危险状态、玩家威胁 |
| 主色-黄 | **御黄** | `#D9A441` | 数值上升、正向趋势、奖励 |
| 主色-蓝 | **缥色** | `#4A7B9D` | 玩家势力、选中态、信息提示 |
| 主色-紫 | **茄皮紫** | `#6E4F6B` | 玩家不可控的外部势力、AI 动向 |
| 文字主 | **黛色** | `#2A2723` | 标题与正文 |
| 文字次 | **烟色** | `#7A6F62` | 次要文字、标签 |
| 底色-面 | **月白** | `#EFE4C9` | 面板底色 |
| 底色-背 | **玄色** | `#1B1815` | 应用背景 |
| 底色-卡 | **米色** | `#F5EAD0` | 卡片/徽章底 |
| 边框 | **驼色** | `#8B6F47` | 面板边框 |
| 装饰 | **描金** | `#A88C3C` | 标题分隔线、关键装饰(用量克制) |

> **与现版本的差异**:现版本是"米黄+古铜"双色调,饱和度偏低,问题是**画面整体偏"温和",危机感弱**。新色板保留米黄底子,把警示色(帝王朱)、趋势色(御黄)、玩家色(缥色)都**提纯**,让地图和 TopBar 的关键状态可以"一眼跳出来"。

### 3.2 字体系统

```css
--font-display: "霞鹜文楷 LXGW WenKai", "Noto Serif SC", "Songti SC", serif;  /* 标题 */
--font-body:    "Noto Serif SC", "Songti SC", "Source Han Serif", serif;     /* 正文 */
--font-num:     "LXGW WenKai Mono", "Noto Sans Mono CJK SC", monospace;      /* 数字徽章 */
--font-label:   "Noto Sans SC", "PingFang SC", "Microsoft YaHei", sans-serif;/* 标签/小字 */
--font-en:      "Linux Libertine", "EB Garamond", "Crimson Text", serif;      /* 英文标注 */
```

**风格决策**:正文用**宋体**(印刷感、稳重),标题用**霞鹜文楷**(手写雕版味),数字用**等宽**(对齐易读),英文/拉丁转写用**Linux Libertine**(V3 用的同款)。

### 3.3 装饰元素（克制的 Art Nouveau 明代化）

V3 用**新艺术运动的植物曲线**(框架+四角+标题抬头的细节),不堆满整个面板。我们要做的:

| 装饰 | 出现位置 | 出现频率 |
|---|---|---|
| **如意云头** 角饰 | 面板四角 | 每个主要面板(1-4 角) |
| **卷草纹** 边线 | 面板左右边框 | 主面板、副面板 |
| **雕花窗格** 顶饰 | 大型面板/弹窗的标题下方 | 全屏事件弹窗、Tutorial 弹窗 |
| **金线分隔** | 标题与内容之间 | 几乎所有面板 |
| **印章 icon** | 关键行动按钮的角标 | "批准奏折"、"宣战"等关键按钮 |
| **水墨晕染** | 应用背景 | 仅 1 处:登录屏/Game Over 屏 |

> 装饰物**必须做成 SVG 组件**,不要用位图。否则缩放时糊。

### 3.4 图标系统（V3 的三分类法,明代化）

| 类别 | 用途 | V3 做法 | 我们的做法 | 实现方式 |
|---|---|---|---|---|
| **A. 详细插画类** | 建筑、商品、产物 | 像迷你插画 | 区域特产(丝绸/瓷器/茶叶/铁矿)、明式建筑(衙门/粮仓/驿所/边关) | **自制 SVG** 16x16 / 32x32 |
| **B. 实体物类** | 事件、科技、人物 | 现实物体(炸药/喇叭) | 奏折、令箭、算盘、玉玺、火铳 | **自制 SVG** 48x48 |
| **C. 简化色块类** | 机制、属性、统计 | 颜色+轮廓 | 国库(铜钱轮廓)、粮(粮袋轮廓)、军(刀轮廓) | **保留 Lucide** 但加**底色块** |
| **D. 派系/势力类** | 派系、势力 | 复杂色编码 | 派系:东林/阉党/浙党…各派系**专属颜色**+ 专属图标(松、竹、梅、菊等文人意象) | **自制 SVG** 24x24 |
| **E. 地形类** | 平原/山地/草原/河流/海岸 | — | 简化的明式山水笔触 | **自制 SVG** 12x12 |

**颜色编码规则**(统一):
- 数值**正向变化**:御黄 `#D9A441`
- 数值**负向变化**:帝王朱 `#B23A2C`
- **选中/玩家**:缥色 `#4A7B9D`
- **危险/濒临崩溃**:帝王朱脉冲(0.8s 周期)
- **可执行行动**:御黄描边

## 五、组件库规范

> 目标:把 `StatBadge` / `Button` / `Panel` / `Tooltip` 全部做成可复用组件,有明确的 API、状态、变体。

### 5.1 按钮(Button)

| 变体 | 用途 | 样式 |
|---|---|---|
| **primary 主行动** | 推进一月、宣战、批准奏折 | 帝王朱底 + 描金边 + 印章角标 |
| **secondary 次行动** | 切 Lens、切 Tab、查看详情 | 月白底 + 驼色边 + 暗色字 |
| **tertiary 弱行动** | 关闭、取消 | 透明 + 暗色字 |
| **danger 破坏** | 撤军、贬谪 | 深朱 + 白字 |
| **gold 高优先级** | 重要决策、关键事件 | 御黄底 + 描金边(用量克制) |

**统一规范**:
- 最小高度 36px(主按钮 44px)
- 三态:default / hover(微亮) / active(微暗) / disabled(灰)
- 图标 + 文字混排时,图标 18px、文字 14px
- hover 时:**关键操作加微动画**(如印章"咔"地盖下)
- 关键操作(战争/重要决策)加**二次确认**

### 5.2 数据徽章(StatBadge v2)

```tsx
<StatBadge
  label="国库"
  value={123000}
  format="万两"          // 自动转 "12.3 万两"
  trend={+15}             // 自动显示 ↑ 御黄
  tone="warning"          // danger / warning / positive / neutral
  showThreshold           // 显示阈值线(粮/军临界线)
  tooltip="..."           // 通用 tooltip
/>
```

**新增能力**:
- **数字滚动动画**(0.3s 平滑过渡,而非突变)
- **趋势箭头**(自动根据 diff 显示)
- **阈值线**(如粮<1万时边框变帝王朱)
- **hover 时展开明细**(例如"国库 12.3 万 = 税入 8.5 万 - 军费 3.2 万...")

### 5.3 面板(Panel)规范

**三种容器**:
1. **抽屉(Drawer)**:右侧滑入,380px 宽,常驻(详情面板)
2. **模态(Modal)**:居中弹出,带遮罩(事件弹窗)
3. **气泡(Popover)**:锚定元素旁,自动定位(Tooltip/选项卡)

**统一结构**:
```
┌─如意云头─┐ ┌─如意云头─┐
╭──────────────────────────╮
│  标题(楷体 18px + 描金分隔)  │ ← 标题区
├──────────────────────────┤
│                          │ ← 内容区
│   (左右内边距 24px)         │
│                          │
├──────────────────────────┤
│  [次按钮]      [主按钮]    │ ← 操作区(可选)
╰──────────────────────────╯
└─如意云头─┘ └─如意云头─┘
```

### 5.4 Tooltip 系统（重要!扩展现有 `FocusTooltip`）

**三种 Tooltip,统一外观**:
- **info**:悬停信息(任何数字、图标、地名)
- **predict**:决策预测(切换 Lens/焦点/军略前的结果预估)
- **detail**:详细机制解释(可展开,可固定)

**统一规范**:
- 出现延时 300ms
- 出现动画 150ms
- 关闭延时 100ms
- 风格:月白底 + 驼色边 + 描金细线
- 内容顶部:标题(楷体加粗)
- 内部:字段 key-value
- 底部:来源/操作提示(可点击)

### 5.5 图标组件(Icon)

| 组件 | 用途 | 示例 |
|---|---|---|
| `IconStat` | 数字+图标 | 国库 12.3万(铜钱图标) |
| `IconBadge` | 带底色块的小图标 | 派系徽章(松+缥色底) |
| `IconButton` | 仅图标的可点击按钮 | 缩放控制、关闭 |
| `IconButtonGroup` | 分段控件 | Lens 切换、军事姿态 |
| `IconStatDelta` | 数字+图标+趋势 | ↑5% 御黄 / ↓3% 帝王朱 |

## 六、信息架构：Lens 系统（最重要的一节）

### 6.1 五个 Lens 的详细定义

#### Lens 1：势力（默认）

- **地图色板**:各势力 `primaryColor` 主导,透明度反映控制度
- **区域 hover 卡**:

## 七、交互细节

### 7.1 键盘快捷键

| 键 | 功能 |
|---|---|
| `Space` | 推进一月 |
| `1` `2` `3` `4` `5` | 切到 Lens 1-5 |
| `Tab` | 循环切换 Lens |
| `Esc` | 关闭弹窗/取消选择 |
| `A` `D` | 上/下一月(预览) |
| `Alt+点击区域` | 居中聚焦 |
| `F` | 切换侧滑面板(打开/关闭) |
| `?` | 显示快捷键帮助 |

### 7.2 动效规范

| 场景 | 动效 | 时长 |
|---|---|---|
| 数值变化 | 数字滚动 | 0.3s |
| 颜色变化 | 渐变 | 0.2s |
| 面板进出 | 滑入+淡入 | 0.25s |
| 重大事件 | 屏幕轻微震动 | 0.4s |
| 新事件 | 邸报飘入 | 0.5s |
| 危机状态 | 红色脉冲边框 | 0.8s 周期 |
| Lens 切换 | 色板过渡 | 0.4s |
| 印章/盖章 | 缩放+轻微旋转 | 0.3s |

**性能约束**:
- 一次最多 10 个元素同时动画
- 装饰元素**默认不动画**,hover 才动
- 尊重 `prefers-reduced-motion`,开启时所有动画降为瞬时

### 7.3 反馈与确认

- 关键操作(开战、改派、贬谪、收税)需**二次确认弹窗**
- 数值变化**即时高亮**:上涨御黄(0.5s 渐隐),下降帝王朱
- 推进月时:右下角快速播放上月"流水账"(可跳过)
- 事件触发时:TopBar 的相应徽章**脉冲**

## 十一、风险与权衡

| 风险 | 缓解策略 |
|---|---|
| 装饰元素拖慢地图渲染 | 所有装饰做成 SVG 且**默认不渲染动画**,zoom 缩小时自动降级 |
| 美术资源开发成本 | 先用 CSS/SVG 制作占位资源,后续可外包精修 |
| 信息密度高,新手劝退 | 渐进解锁 + Tutorial + Lens 切换降低单屏信息量 |
| 改 UI 影响既有数值 | UI 改动只动 `ui/` 和 `app/App.css`,`core/` 不动 |
| 测试用例缺失 | Phase 6 集中补 vitest + Testing Library 用例 |
| 色盲玩家体验 | "图案+颜色"双重编码,Phase 6 验证 |

## 十二、决策记录（建议你拍板的几个分叉）

> 这些点 V3 都给了一个明确选择,但你的明朝主题可能要走不同的路。我给建议,但你定。

### D1. 信息密度 vs. 简洁

- **V3**:极简地图 + 信息密集面板(玩家进面板读详情)
- **建议**:沿用 V3 路线 —— 地图保持简洁,详情全在右侧滑面板

### D2. 朝代叙事 vs. 玩法严肃

- **V3**:Journal 是"时代叙事",读起来像史书,有真实感
- **建议**:邸报要做得**有明代官话味道**,但别学某些国产游戏搞"搞笑古风",保持史料感

### D3. 默认聚焦还是全局视角

- **V3**:默认地图全局,玩家主动点选区域
- **建议**:**跟随玩家首都默认居中 + 缩放级别 1.2x**(突出玩家),让玩家"身在明帝国"

### D4. Tutorial 强度

- **V3**:主动 + 5 折引导 + 信息图
- **建议**:**首次必看**(不能跳过),后续可"查看历史奏折"再读

### D5. 是否做暗色主题

- **V3**:无暗色主题(只有"夜间",即整个游戏画面暗下来)
- **建议**:**先不做暗色主题**。万历朝就是黄昏色调,符合时代感;做暗色反而要重新配一套

## 写在最后

V3 之所以是策略游戏的 UI 标杆,不是因为它"华丽",而是因为它**让复杂游戏的复杂度不外溢到玩家眼前**。我们这版方案的核心目标也是这个:

> 玩家不应该被"看见"系统有多复杂,而应该被"卷入"明代的朝堂与战场。

剩下的就是写代码了。从 Phase 1 开始,3 个月内能见到明显效果。
