## 0. 摘要

v0.4 SPEC 提出的 P0–P3（模拟正确性、账本、人口群体、产业市场）在代码层面已落地，但一次全量回归暴露出**底层模拟在系统集成后数值全面崩溃**：30 年模拟大明人口归零、国库亏到 -1100 万、粮储 -1.09 亿、粮价涨到 4×10¹⁷。

本轮工作分两段：
- **第一段（已完成）——全量回归与 fix**：定位并修复了 7 类数值缺陷，使 20×120 批量模拟从"全面崩溃"变为"数值健康可解释"。
- **第二段（本文档主线）——向维多利亚3闭环演进**：在数值稳固的地基上，打通当前断裂的"人口—生产—市场—政治—外交—战争"闭环。

**核心论断**：当前项目距"大明版维多利亚3"的最大差距，不是系统数量不足，而是**已建子系统之间没有形成维多利亚3式的单一驱动闭环**。下一阶段的最高优先级是"接通闭环"，而非继续新增孤岛系统。

## 3. 距维多利亚3的核心差距：闭环断裂

维多利亚3的引擎本质是一条单一闭环：

```
pop 劳动 → 产业生产 → 市场供需/价格 → pop 消费/生活水平
   → 政治力量/利益集团 → 法律/制度 → 反作用于生产
   → 外交博弈 → 战争动员/战线 → 破坏生产与人口 → 回到起点
```

当前项目每一环都建了**零件**，但环与环之间的**齿轮没有咬合**：

1. **经济环断裂**：`market`（产业商品）与 `economy`（pop 口粮/税收）是两套粮食系统。产业产出的粮食不进入 pop 消费，pop 的需求不真正驱动价格形成。
2. **生活环断裂**：popGroups 的 `needsSatisfaction` 由 `grainPerCapita` 直接决定，不经过市场价格；wealth 不积累、不分化。
3. **政治环断裂**：clique 力量来自地区属性映射，不来自 pop 财富/官职；support 变化不驱动任何政策后果。
4. **修正环断裂**：modifier 是死数据，事件/法律写下的长期影响实际不生效——这使"低操作高后果"的设计意图落空。
5. **外交/战争环断裂**：战争是单次战斗，不消耗动员/补给/财政的长期积累，也不经外交博弈前置。

**结论**：下一阶段不是"再建 5 个新系统"，而是"把这 5 条断线接通"。

## 4. 下一阶段实施规格（S1–S6）

按"闭环依赖"而非"系统数量"排序。每一阶都要求：先打通齿轮、再补内容，且每阶必须保持 batch 20×120 健康与测试全绿。

### S1　激活修正系统与账本驱动（接通"后果"环）

**目标**：让事件/法律/灾害的长期修正真正生效，让账本成为财政的唯一真相源。

- 改造 `aggregateModifierEffect`，在税收、行政、稳定、控制、军组、生产等关键计算点查询并应用修正（按 scope: global/faction/region）。
- 让 `ledger` 成为 `treasury`/`grainReserve` 的唯一驱动：所有收支走账本条目，月末 `applyLedgerToState` 结算，移除 `simulation` 中的散点加减。
- 接入开发调试面板（SPEC §18.3）：查看任一数值的全部修正来源与最终公式。

**验收**：事件写入的修正能在下月税收/稳定上可观测地生效；移除散点加减后国库变化与账本净额一致（SPEC §21.2 不变量"国库变化等于账本净额"）。

### S2　市场—人口—生产整合（接通"经济"环）

**目标**：统一粮食/经济系统为单一闭环。

- 废除 `economy.ts` 中绕过市场的口粮直算；产业产出粮食 → 进入市场 supply → pop 按需求购买（需求由 pop 类型/规模/生活水平驱动）→ 价格形成 → 剩余进 `grainStock`。
- popGroups 的 `needsSatisfaction` 改为"能否以市场价格买得起需求篮子"，引入 wealth 积累与分化。
- 引入白银货币约束：pop 购买、军饷、建设都消耗白银，白银短缺传导为购买力下降（SPEC §8.6）。
- 产业 ownership 的利润流向对应 pop 群体与政治力量（SPEC §8.2）。

**验收**：灾荒/战争推高粮价 → pop 购买力下降 → needsSatisfaction 下降 → 激进度上升，形成可读因果；江南与辽东经济结构明显不同。

### S3　利益集团政治力量来源（接通"社会—政治"环）

**目标**：让政治力量来自真实社会基础。

- clique strength 改由其代言 pop 群体的 wealth + 人口 + 官职 + 土地（产业 ownership）聚合，而非地区属性映射。
- 利益集团持有 `preferredLaws`/`opposedLaws`，其 approval 由当前法律、预算分配、成员生活水平、税负、任用/打压驱动。
- 高 strength + 低 approval 的集团推动政治运动（减税/开海/军饷/自治等）。

**验收**：玩家加税 → 农民/士绅集团 approval 下降 → 推动减税运动；不同政权拥有不同集团结构。

### S4　法律与改革系统（接通"制度"环）

**目标**：改革是带阻力、可失败、有长期路径的过程。

- 实现 SPEC §11 的法律类别（税制/土地/军制/海贸/地方治理/财政权力）。
- 改革流程：提出 → 计算支持/反对集团 → 进入进度 → 辩论/拖延事件 → 妥协/失败/通过 → 按行政能力逐步落实（落实即写入 S1 的 modifier）。
- 改革速度由合法性、行政、皇权、集团力量、控制度、战争、腐败决定。

**验收**：一条鞭法可能巩固、妥协、停滞或遭清算；改革产生明确受益者与反对者。

### S5　外交博弈与持续战争（接通"外交—战争"环）

**目标**：战争是财政、补给、动员、外交、国内政治的综合结果。

- `DiplomaticRelation`（relation/trust/threat/rivalry/obligations/truce）+ 条约（同盟/朝贡/互市/附庸/停战）。
- 外交博弈阶段：提要求 → 拉盟友 → 威胁评估 → 动员集结 → 让步/妥协/开战。
- 战线 `FrontState`：玩家选主战区/姿态/动员规模/补给优先级，每月按兵力×组织×补给×地形推进。
- 和平谈判：按战争支持、占领、战损、财政、国内政治结算，不止"占领即吞并"。

**验收**：战争不再由单月攻击决定；军费/补给/国内政治能迫使停战；AI 能备战与让步。

### S6　历史局势与完整周期（内容收口）

**目标**：把孤立事件升级为系统驱动的长期局势。

- `SituationState`（stage/progress/variables/outcomes）承载张居正改革、三大征、建州统一、辽东危机、陕西流民、南明分裂等。
- 局势变量由 S1–S5 的系统状态推动；事件作为局势的叙事表现。
- 补齐 1573–1662 主线，目标多结局可重复出现。

**验收**：历史局势由系统条件推动；完整周期可运行；多种结局在批量模拟中出现。

## 5. 优先级与依赖

```
S1（修正+账本）   ← 所有后续系统的生效基础，必须最先
   ↓
S2（市场整合）    ← 经济闭环，S3/S4 的财富/税负来源
   ↓
S3（政治力量）    ← 依赖 S2 的 wealth 聚合
   ↓
S4（法律改革）    ← 依赖 S3 的集团支持/反对
   ↓
S5（外交战争）    ← 依赖 S2 财政/S4 制度
   ↓
S6（历史局势）    ← 依赖 S1–S5 全部，内容收口
```

**本轮（v2）实施范围建议**：S1 + S2（接通后果环与经济环），这是性价比最高、解锁后续一切的一步。S3–S6 在其稳固后再逐阶推进。

### 9.3 闭环达成

第二条闭环接通：pop 财富（S2）→ 集团 strength → approval（生活水平+税负+政策）→ 政治运动 → S1 modifier 反作用于经济/财政/控制 → 回到起点。政治不再是装饰，而是会真实削弱或巩固政权的力量。

### 9.4 设计权衡

- **strength 分布**：缙绅 clique（peasant+tenant 基础）strength~86 主导，东林（gentry pop）~10。这是"乡村强权"模型——明末政治动荡确以乡村（流民/佃变/士绅自治）为基础，符合"皇权不下县"。
- **运动是危机放大器**：让步强度极温和（control-flat −1 等），无危机时几乎无感（seed1 零运动大明健康扩张 16→27 区），战争/灾荒压低生活水平时才触发并温和放大。
- **approval 与 support 正交**：support 驱动 administration（不变），approval 驱动运动。强而不满的集团推动运动，运动让步回升 support。
