> 目标版本：从当前轻量历史推演 MVP，升级为具有“人口—经济—政治—外交—战争”闭环的晚明社会模拟大战略游戏

---

## 0. 文档摘要

《万历：山河崩塌》当前版本已经具备月度推进、人口、粮食、财政、战争、叛乱、事件、AI、地图信息层和存档等基础能力，能够完成一局可运行、可复现的历史地图推演。

下一阶段不应继续以“增加地图、事件、插画和势力数量”为主要目标，而应转向底层模拟质量建设。项目应借鉴《维多利亚3》的核心设计方法，但不照搬工业时代题材和高微操结构，而是建立符合晚明历史背景的社会经济系统：

```text
人口与身份
→ 劳动力、赋役和消费
→ 农业、手工业、矿业、商贸生产
→ 商品供需、价格和区域流通
→ 税粮、白银、官俸、军饷和国家预算
→ 生活压力、财富分化和政治力量
→ 派系支持、制度改革、地方自治和民变
→ 行政效率、外交能力、动员和战争
→ 反向影响人口、生产与社会秩序
```

本方案的核心目标不是增加玩家操作量，而是让玩家继续通过少量高层决策，影响更复杂、更可解释的社会系统。

# 1. 项目重新定义

## 1.1 产品定位

《万历：山河崩塌》是一款以 1573—1662 年中国及东北亚为主要舞台的单机 Web 历史社会推演游戏。

玩家选择一个历史势力，通过有限的高层战略决策，影响人口结构、土地生产、区域贸易、国家财政、利益集团、制度改革、外交关系、军事动员和政权兴衰。

游戏不强调单位微操，不要求玩家逐城建设或逐支部队移动，而是强调：

- 长期政策的累积后果；
- 社会结构对国家能力的限制；
- 财政、粮食、政治与战争之间的相互制约；
- 历史压力下的多种可能结局；
- 每一次数值变化都能被解释。

## 1.2 一句话规格

> 玩家以少量战略决策推动晚明国家治理，让人口、产业、财政、政治和战争系统自动演化，并在历史危机中改变中国及东北亚的政权格局。

## 1.3 目标体验

玩家应当逐步形成以下认知：

1. 国家强大不等于国库充足。
2. 人口众多不等于可征税人口多。
3. 军队规模不等于真实战斗力。
4. 政策收益会被既得利益集团、腐败、交通和控制度削弱。
5. 战争不仅消耗兵力，还会破坏生产、贸易、财政和政治稳定。
6. 历史事件不是孤立弹窗，而是社会矛盾积累到一定程度后的叙事表现。
7. 玩家改变的不是一个数字，而是长期制度路径。

# 2. 设计原则

## 2.1 核心设计原则

1. **低操作，高后果**  
   玩家操作保持精简，但每个决策必须影响多个系统，并在数月或数年后显现后果。

2. **系统先于事件**  
   事件用于表现系统变化，而不是替代系统本身。

3. **人口是国家能力来源**  
   税收、军队、劳动力、消费、政治力量和叛乱都必须追溯到人口群体。

4. **所有资源必须有来源与去向**  
   粮食、白银、人口、兵力、政治力量不得凭空产生或消失。

5. **玩家与 AI 使用同一套规则**  
   AI 可以拥有不同性格和历史目标，但不得使用隐藏资源或跳过系统限制。

6. **复杂计算必须可解释**  
   UI 必须展示数值来源、主要修正、趋势和风险。

7. **历史方向可辨认，历史结果不强制**  
   历史事件和历史诉求提供压力，但不锁死真实历史结局。

8. **数据驱动，逻辑解耦**  
   势力、地区、商品、产业、派系、制度、事件和 AI 目标应尽量配置化。

9. **固定种子与可复现性**  
   同一初始状态、同一决策、同一随机种子必须得到相同结果。

10. **优先保证长期模拟稳定**  
    在增加内容前，必须先通过 10 年、30 年和完整历史周期的批量模拟。

## 2.2 明确不做

以下内容不作为中期开发重点：

- 单支部队自由移动；
- 武将 RPG 养成；
- 完整家族婚姻系统；
- 复杂宫斗角色关系网；
- 3D 地图；
- 实时战斗；
- 每个城市单独建设；
- 过度细分的兵种装备生产；
- 多人联机；
- 完整复制《维多利亚3》的全部商品和人口类别。

# 3. 当前版本基线

## 3.1 已有能力

当前仓库已具备：

- React + TypeScript + Vite；
- Zustand 状态管理；
- 固定随机种子；
- 月度推进；
- 地区人口与承载力；
- 粮食、税收、国库与军队维护；
- 地区控制度、稳定度、叛乱压力；
- 简化战争结算；
- 历史事件与选项；
- 势力 AI；
- 地图信息层；
- 派系支持度；
- IndexedDB 存档；
- 批量模拟脚本；
- Vitest 测试体系；
- 教程、帮助、快捷键、侧边栏和事件插画。

## 3.2 当前结构的主要限制

### 3.2.1 人口过度聚合

每个地区只有一个人口总数，无法表达：

- 谁在生产；
- 谁承担赋税；
- 谁拥有土地；
- 谁掌握政治力量；
- 谁在饥荒中首先破产；
- 谁会成为流民或叛军。

### 3.2.2 经济缺少生产与市场

当前经济主要通过人口乘以系数直接生成粮食和税收，缺少：

- 产业设施；
- 劳动力分配；
- 商品投入与产出；
- 商品价格；
- 区域供需；
- 运输损耗；
- 商贸路线；
- 工资和生活水平。

### 3.2.3 政治集团缺少社会基础

当前派系支持度与地区经济属性存在关联，但没有真正绑定：

- 人口规模；
- 财富；
- 土地；
- 官职；
- 军队；
- 商业网络；
- 地方控制权。

### 3.2.4 战争不是持续状态

当前战争更接近“每月一次独立战斗”，缺少：

- 战线；
- 动员；
- 补给；
- 累计进度；
- 战争目标；
- 和平谈判；
- 长期财政和社会影响。

### 3.2.5 事件与长期系统连接不足

事件可以写入长期修正，但当前月度模拟需要建立统一的修正消费、递减和到期机制。

### 3.2.6 AI 缺少长期规划

当前 AI 主要基于即时危机评分选择目标，缺少：

- 长期国家目标；
- 中期计划；
- 政策惯性；
- 外交记忆；
- 战争准备；
- 资源储备计划。

## 4.2 状态分层

游戏状态应划分为四类：

### A. 永久基础状态

- 地区地形；
- 气候；
- 地理连接；
- 初始文化和政权核心；
- 基础农业潜力；
- 基础矿产和港口条件。

### B. 持久可变化状态

- 人口群体；
- 产业设施；
- 地区财富；
- 地区控制度；
- 政治集团力量；
- 法律制度；
- 外交关系；
- 战争状态。

### C. 临时修正状态

- 灾害；
- 战争破坏；
- 政策运动；
- 事件修正；
- 季节修正；
- 短期军费；
- 临时税赋。

### D. 派生状态

派生状态每月重新计算，不作为永久基础值写回：

- 有效行政能力；
- 有效税收效率；
- 有效军事组织；
- 有效合法性；
- 总市场价格；
- 生活水平；
- 利益集团政治力量；
- 战线补给率。

# 5. 月度结算流水线

## 5.1 强制结算顺序

```text
01. 月初状态准备
02. 到期修正清理
03. AI 更新长期计划和月度决策
04. 人口需求计算
05. 劳动力与就业分配
06. 产业生产
07. 商品市场与价格结算
08. 人口消费与生活水平结算
09. 人口增长、死亡、迁移和流民化
10. 地区税赋与国家财政结算
11. 官仓、地方粮仓和中央储备调度
12. 政治力量、集团支持与政治运动结算
13. 行政、腐败、控制度和地方自治结算
14. 外交关系与外交博弈结算
15. 动员、补给、战线和战役结算
16. 战争破坏、叛乱与动态势力生成
17. 历史局势和事件检测
18. 报告、趋势、警报和历史记录生成
19. 状态不变量校验
20. 推进日期
```

## 5.2 结算原则

- 人口消费必须发生在生产和市场结算之后；
- 税收必须基于已经完成的生产、收入和控制度；
- 军事补给必须从可用粮食、白银和运输能力中扣除；
- 政治反应必须基于当月生活水平、税负和政策变化；
- 历史事件必须在所有系统结算后检测；
- 临时修正不能写回永久基础值；
- 同一资源不得在地区库存和国家库存重复记账。

# 7. 人口系统 SPEC

## 7.1 首版人口类别

第一阶段只实现八类人口：

| 类型 | 主要功能 | 主要风险 |
|---|---|---|
| 自耕农 | 粮食生产、基础税源 | 灾荒、兼并 |
| 佃户 | 农业劳动力 | 高租、高税、流民化 |
| 工匠 | 手工业生产 | 原料价格、市场萎缩 |
| 商人 | 商贸和资本积累 | 海禁、战争、税监 |
| 士绅 | 土地、地方治理、政治力量 | 改革和清丈阻力 |
| 官吏 | 行政能力 | 俸禄不足、腐败 |
| 军户/兵丁 | 军事动员 | 军饷拖欠、逃亡 |
| 流民 | 无稳定生产 | 饥荒、迁徙、叛乱 |

## 7.2 人口变化

```text
下月人口 =
当前人口
+ 出生
- 普通死亡
- 饥荒死亡
- 疫病死亡
- 战争死亡
+ 净迁移
+ 身份转化
```

身份转化示例：

- 佃户破产 → 流民；
- 流民获得土地 → 自耕农；
- 自耕农失地 → 佃户；
- 军队复员 → 军户或农民；
- 工坊扩张 → 农民转化为工匠；
- 科举和官僚扩张 → 士绅或官吏增长。

## 7.3 生活需求

基础需求：

- 粮食；
- 布匹；
- 盐；
- 住房抽象值。

高级需求：

- 茶叶；
- 瓷器；
- 奢侈消费；
- 教育和文化支出。

生活水平不是单一财富值，而由以下因素计算：

```text
需求满足度
+ 收入稳定性
- 税负
- 地租
- 商品价格压力
- 战争破坏
- 灾害压力
```

## 7.4 流民机制

流民不是普通人口减少，而是独立社会力量。

流民来源：

- 饥荒；
- 土地兼并；
- 战争占领；
- 税负过重；
- 产业破产；
- 地区控制崩溃。

流民结果：

- 降低生产；
- 增加迁徙；
- 提高治安成本；
- 增加地方豪强和叛军兵源；
- 形成跨地区流动；
- 在特定条件下生成农民军。

# 8. 生产与经济系统 SPEC

## 8.1 生产逻辑

每个产业的实际产出：

```text
基础产出
× 设施等级
× 劳动力满足率
× 输入品满足率
× 地区生产潜力
× 稳定度
× 控制度
× 技术/制度修正
× 灾害与战争修正
```

## 8.2 所有权

产业所有权影响利润流向和政治力量：

- 国家所有：利润进入国库；
- 士绅所有：强化士绅财富与政治力量；
- 商人所有：强化商人财富和开放贸易诉求；
- 军事所有：强化将领和军镇独立性；
- 社区所有：收益分散到当地人口，政治集中度较低。

## 8.3 投资机制

玩家不逐个点击建设，而是设置国家发展重点：

- 农业恢复；
- 水利和粮仓；
- 商贸开放；
- 矿业开发；
- 军镇建设；
- 交通整顿；
- 手工业扶持。

系统根据以下因素自动分配投资：

- 国家预算；
- 地区潜力；
- 政策优先级；
- 政治集团阻力；
- 地区控制度；
- 当前战争压力；
- AI 中期计划。

## 8.4 市场规模

不实现全球统一市场，采用“区域市场网络”。

建议初期市场：

- 北方京畿市场；
- 江南白银与手工业市场；
- 湖广粮食市场；
- 西南山地市场；
- 辽东边疆市场；
- 蒙古草原市场；
- 朝鲜市场；
- 日本市场。

地区通过道路、水运、港口和外交关系连接。

## 8.5 价格

商品价格按供需计算，但必须设置平滑机制：

```text
目标价格 = 基准价格 × 需求 / 供给
本月价格 = 上月价格 × 0.7 + 目标价格 × 0.3
```

应限制单月涨跌幅，避免价格振荡。

## 8.6 白银机制

晚明财政应突出白银的重要性。

白银来源：

- 商税；
- 海贸；
- 日本和海外输入；
- 矿业；
- 赋税折银；
- 战争赔款。

白银短缺结果：

- 实际税收下降；
- 粮价与银价错配；
- 官俸、军饷支付困难；
- 农民纳税压力增加；
- 改革和加派需求上升。

# 9. 财政与粮食系统 SPEC

## 9.1 三层粮食库存

粮食必须分为：

1. 民间粮食；
2. 地方官仓；
3. 中央可调度储备。

粮食征收和运输流程：

```text

→ 人口消费
→ 地租与市场交易
→ 地方税粮
→ 地方官仓
→ 运输损耗
→ 中央粮储或前线补给
```

## 9.2 国家财政

收入：

- 田赋；
- 商税；
- 盐课；
- 关税；
- 朝贡和属国收入；
- 临时加派；
- 战争赔款；
- 国家产业收入。

支出：

- 官僚俸禄；
- 常备军军饷；
- 战时补给；
- 赈灾；
- 工程建设；
- 外交维护；
- 债务利息；
- 宫廷和宗室开支；
- 事件支出。

## 9.3 赤字与债务

财政赤字不得立即判定破产，应经历：

1. 动用国库储备；
2. 拖欠官俸；
3. 拖欠军饷；
4. 向商人和士绅借债；
5. 临时加派；
6. 出售特权；
7. 财政信用崩溃。

不同方式影响不同集团：

- 向商人借债：提升商人政治力量；
- 向士绅借债：提高地方自治和减税要求；
- 拖欠军饷：提高哗变风险；
- 临时加派：提高农民与流民激进度。

月度结余      -50,000

# 13. 军事系统 SPEC

## 13.1 军事资源

军队战斗力由以下因素决定：

- 可动员人口；
- 兵员类型；
- 组织度；
- 训练；
- 军饷支付率；
- 粮食补给率；
- 武器和马匹供给；
- 将领集团支持；
- 地形；
- 季节；
- 战争疲劳；
- 前线距离；
- 道路和水运。

## 13.2 动员

动员会：

- 将军户、农民或部族人口转为兵丁；
- 降低地方生产；
- 增加军费和粮食消耗；
- 提高将领集团力量；
- 长期动员导致逃亡和流民化。

## 13.3 战线

每场战争根据地理连接生成战线。

玩家不控制单位，只选择：

- 主战区；
- 攻守姿态；
- 动员规模；
- 补给优先级；
- 是否增援。

## 13.4 战役

每月战役结算：

```text
投入兵力
× 有效组织度
× 军饷支付率
× 补给率
× 地形修正
× 指挥修正
× 随机波动
```

结果影响：

- 伤亡；
- 战线优势；
- 占领进度；
- 地区破坏；
- 战争支持；
- 财政；
- 粮食；
- 政治集团态度；
- 流民数量。

## 13.5 和平

和平谈判依据：

- 战争支持；
- 占领地区；
- 战损；
- 财政压力；
- 国内政治；
- 盟友态度；
- 战争目标成本。

不得只有“占领即吞并”。

# 15. AI 系统 SPEC

## 15.1 三层 AI

- 储备粮食；
- 整顿财政；
- 推动改革；
- 准备战争；
- 修复地区；
- 拉拢政治集团；

- 削弱目标势力。

- 选择发展重点；
- 分配预算；

- 投入战线；
- 处理政治运动；

AI 只有在以下条件满足时才更换主要政策：

```text

```

## 21.3 批量模拟

### 10 年模拟

目标：

- 无崩溃；
- 无指数爆炸；
- 商品价格不过度振荡；
- 大部分势力仍具有合理财政。

目标：

- 至少出现不同历史路径；
- 建州不应必然崛起或必然灭亡；
- 明朝不应永远稳定；
- 流民、财政和战争形成可读因果。

### 完整周期模拟

1573—1662：

- 可产生明朝延续；
- 可产生后金/清崛起；
- 可产生地方割据；
- 可产生农民军政权；
- 可产生南明或替代继承政权；
- 结果分布不能被单一脚本锁死。

# 25. 最终验收标准

项目达到“接近《维多利亚3》式完成度”的最低标准，不是系统数量相同，而是满足以下条件：

1. 国家能力来自人口和经济，而不是直接配置数值。
2. 商品供需和价格能够影响生活、税收、军费和政治。
3. 政治集团力量来自真实社会基础。
4. 改革会产生明确受益者、反对者和长期路径。
5. 战争是财政、补给、动员、外交和国内政治的综合结果。
6. 历史事件由局势和系统状态驱动。
7. AI 能够围绕长期目标制定准备计划。
8. 玩家可以通过 UI 解释主要数值变化。
9. 同一随机种子和决策可以复现结果。
10. 完整历史周期能产生多个合理且不同的结局。

# 26. 结论

《万历：山河崩塌》的下一阶段重点，不应是继续增加表层内容，而应完成底层闭环。

最关键的开发顺序是：

```text
修复模拟正确性
→ 建立账本和数值解释
→ 建立人口群体
→ 建立生产与市场
→ 建立社会政治力量
→ 建立外交和持续战争
→ 用历史局势组织完整内容
```

项目应保持自身特色：

- 不复制《维多利亚3》的工业时代设定；
- 不复制其高密度操作；
- 借鉴其系统互相驱动的设计方法；
- 形成一套属于晚明历史的国家与社会模拟系统。

最终目标不是“做一个小型《维多利亚3》”，而是：

> 做一款操作更少、历史主题更集中、因果链更清晰、能够长期推演晚明国家崩塌与重建过程的社会模拟大战略游戏。
